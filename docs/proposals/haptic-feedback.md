# 实现方案：统一触感反馈 (Haptic Feedback)

**日期**: 2026-02-02
**状态**: ✅ 已完成
**目标**: 抽离并统一全站的设备振动（Vibrate）逻辑，提升交互的物理感知体验。

## 1. 实现概述

本方案已全面实施，通过抽象核心工具类和 React Hook，实现了全站点击、成功反馈等交互行为的触感统一。

## 2. 技术细节

### 2.1 核心工具 (`src/lib/haptic.ts`)
定义了标准化的振动预设：
- `light`: 10ms (基础点击、关闭弹窗)
- `medium`: 20ms (中等反馈)
- `success`: [10, 30, 10] (操作成功，如复制、点赞)
- `warning`: [50, 100, 50] (警告/错误)

### 2.2 React Hook (`src/lib/hooks.ts`)
封装了 `useHaptic` Hook，提供简单的 `vibrate(pattern)` 调用方式，并内置了环境安全检查。

## 3. 落地场景

### 3.1 `AboutProfile` 组件
- **链接点击**: 所有社交链接及朋友链接点击均触发 `light` 反馈。
- **弹窗交互**: 打开及关闭 Modal 均触发 `light` 反馈。
- **内容复制**: Email 及 RSS 地址复制成功后触发 `success` 节奏反馈。

### 3.2 导航组件
- **`BottomNavigation`**: 移动端导航项切换统一使用 `light` 反馈。
- **`PostBottomBar`**: 返回、滚动至评论、分享、点赞点击均使用 `light` 反馈；点赞成功/复制成功使用 `success` 反馈。

## 4. 收益总结
- **体验一致性**: 访客在不同页面、不同组件间的操作反馈节奏完全相同。
- **代码整洁度**: 移除了散落在各处的硬编码 `navigator.vibrate` 逻辑。
- **安全性**: 统一处理了 SSR 环境及浏览器兼容性检查。
