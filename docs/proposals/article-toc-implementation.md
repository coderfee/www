# 文章阅读体验升级：目录 (TOC) 实施方案

**日期**: 2026-02-02
**状态**: ✅ 已完成
**目标**: 为长文章提供清晰的结构导航，提升桌面端的沉浸式阅读体验与移动端的快速跳转效率，同时保持界面极简风格。

---

## 1. 核心设计哲学

**"双端异构，无感接入"**

- **Desktop**: 利用宽屏优势，采用 **右侧悬浮 (Sticky Sidebar)** 布局。不仅不干扰正文阅读，反而作为视觉平衡点存在。
- **Mobile**: 考虑到屏幕空间宝贵，采用 **悬浮气泡 (Popover Menu)** 交互。将其收纳进现有的 `PostBottomBar` 体系中，按需唤起。

---

## 2. 桌面端交互设计 (Desktop)

### 2.1 布局策略
- **位置**: 独立于主文章容器（`max-w-3xl`）之外，固定在视口右侧中部 (`fixed right-8 top-1/2`)。
- **响应式**: 仅在 `xl` (1280px+) 屏幕上显示，确保主内容区不被挤压。

### 2.2 视觉风格
- **极简圆点**: 默认仅显示一列垂直的圆点，视觉干扰极低。
    - **H2**: 标准圆点 (`w-1.5`)。
    - **H3**: 小圆点 (`w-1`)，与主层级形成对比。
- **回到顶部**: 顶部集成统一风格的“向上箭头”按钮，作为列表的视觉锚点。
- **Tooltip**: 鼠标悬停时弹出黑色气泡显示完整标题。

### 2.3 交互细节
- **Active 高亮**: 当前阅读章节自动变长条 (`h-6`)，其余章节保持圆点。
- **对齐**: 引入统一的 `w-5` 容器，确保所有元素（箭头、大点、小点）的几何中心严格垂直对齐。

---

## 3. 移动端交互设计 (Mobile)

### 3.1 入口整合
- **位置**: 复用底部的 `PostBottomBar` 组件。
- **触发**: 在“返回”与“评论”之间新增一个 **“目录”图标按钮**。

### 3.2 交互形式
- **Popover Menu**: 放弃全屏抽屉，改为更轻量的悬浮菜单。
- **位置**: 悬浮在底部导航栏上方 (`bottom-24`)，宽度适中 (`max-w-xs`)。
- **视觉**:
    - **材质**: 高斯模糊 (`backdrop-blur-2xl`) + 半透明背景，类似 iOS Context Menu。
    - **动画**: 原点缩放 (`scale` + `opacity`)，轻盈灵动。
- **功能**:
    - 点击任意章节立即关闭菜单并平滑滚动。
    - 顶部集成“回到顶部”快捷入口。
    - 支持点击背景快速关闭。

---

## 4. 技术实施方案

### 4.1 数据流
- 利用 Astro `render(post)` 返回的 `headings` 数组透传给 React 组件。

### 4.2 组件架构
`src/components/blog/toc/` 目录：

1.  **`TableOfContents.tsx` (Desktop)**
    - 负责渲染右侧悬浮圆点列表。
    - 包含 Tooltip 和滚动监听逻辑。

2.  **`MobileTOCDrawer.tsx` (Mobile)**
    - 负责渲染移动端 Popover 菜单。
    - 使用 `React Portal` 渲染至 `body` 根节点，解决层级问题。

3.  **`useScrollSpy.ts` (Shared Hook)**
    - 封装核心滚动监听逻辑 (`window.scrollY` + `getBoundingClientRect`)。
    - 实现了“寻找视口顶部最后一个标题”的算法，确保状态精准。

---

## 5. 总结
本方案通过差异化的交互设计，完美适配了双端场景：桌面端提供了一种“不打扰”的伴随式导航，而移动端则提供了一种“即用即走”的高效工具。
