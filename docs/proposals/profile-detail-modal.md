# 实现方案：个人画像详情弹窗 (Profile Detail Modal)

**日期**: 2026-02-02
**状态**: ✅ 已完成
**目标**: 为集成式个人名片增加深度交互，通过弹窗展示博主的全方位画像（技术、创作、兴趣）。

## 1. 方案背景

本功能已全面上线。通过将个人信息“名片化”并增加详情弹窗，成功将博主的“码龄”、“创作量”以及“核心支柱领域”进行聚合展示，建立了更立体、更具信任感的个人品牌形象。

## 2. 最终设计

### 2.1 物理级交互
- **触发**: 点击网格首项（集成名片），触发 `light` 触感反馈并唤起弹窗。
- **动画**: 使用 `layoutId="profile-modal"` 建立 Shared Element Transition。通过源卡片与 Modal 容器间 `transition` 参数的严格同步，实现了丝滑的物理形变效果。
- **关闭**: 点击背景遮罩或底部悬浮的圆形 [X] 按钮。

### 2.2 UI 结构 (紧凑型规范)
- **Header**: 放大版头像（保留动态在线状态）+ 姓名 + Slogan（在代码与文字间寻找平衡）。
- **Stats Grid (四宫格)**:
    - **码龄**: `9+ 年`
    - **原创博文**: `60+ 篇`
    - **状态**: `在线中` (绿字反馈)
    - **坐标**: `北京`
- **支柱领域 (标签云)**:
    - **编程**: `Astro`, `React`, `TypeScript`, `Python`
    - **阅读**: `技术`, `科幻`, `哲学`, `小说`
    - **写作**: `记录思考`, `生活在别处`

## 3. 技术实现回顾

### 3.1 状态管理
扩展 `ModalType` 联合类型以支持 `profile` 模式。

### 3.2 布局规范
- 弹窗采用四周等宽的 `p-8` 内边距。
- 内容区块通过 `space-y-6` 保持一致的节奏。
- 关闭按钮通过 `-bottom-16` 彻底脱离卡片容器，大幅降低误触率。

## 4. 收益总结
该功能的实施不仅增强了页面的视觉表现力，更通过“支柱领域”的展示补全了博主的人设画像，使 `About` 页面从简单的链接列表进化为一个具有深度的个人展示空间。
