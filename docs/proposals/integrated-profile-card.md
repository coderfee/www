# 实现方案：集成式个人名片卡片 (Integrated Profile Card)

**日期**: 2026-02-02
**状态**: 提议中
**目标**: 将传统的顶部页眉重构为网格系统中的一个“特写卡片”，提升页面的整体感与交互现代感。

## 1. 方案背景

目前 `AboutProfile` 页面采用传统的“头像 + 标题 + 网格链接”布局。虽然清晰，但在追求极致紧凑和交互感的语境下，顶部的独立区域显得略微厚重。通过将其“卡片化”并融入网格，可以使整个页面变成一个纯粹的、高度一致的“交互矩阵”。

## 2. 详细设计

### 2.1 布局调整
- **网格入口**: 个人名片将作为网格的第一个元素。
- **跨列展示**: 在桌面端（`md`），名片将占据两列宽度（`col-span-2`）；在移动端，保持单列宽度以维持一致的节奏。
- **视觉风格**: 
    - 沿用现有社交链接卡片的玻璃拟态背景。
    - 左侧展示中等尺寸的头像（保留在线状态点）。
    - 右侧垂直排列姓名与副标题（Bio）。

### 2.2 交互增强
- **Hover 反馈**: 与其他社交卡片一致，Hover 时带有轻微缩放和背景加深。
- **触感反馈**: 点击时触发 `light` 模式振动。
- **潜在扩展**: 点击该名片可弹出一个更详细的“个人自传”Modal（可选）。

## 3. 技术实现逻辑

### 3.1 数据结构
将个人信息定义为一个特殊的项，并在渲染时通过条件判断进行跨列处理：

```tsx
// 逻辑示例
{socialLinks.map((link, index) => (
  <motion.div
    className={index === 0 ? "md:col-span-2" : "col-span-1"}
    // ... 其他属性
  >
    {/* 内容渲染 */}
  </motion.div>
))}
```

### 3.2 布局微调
由于名片跨列，可能需要微调 `grid-cols-2` 的间距和容器的 `max-w`，以确保整体视觉比例在高度集成后依然稳固。

## 4. 预期收益
- **视觉冲击力**: 开篇即是一个精致的大卡片，相比传统的页眉更具吸引力。
- **空间利用率**: 极大缩减了原本顶部的留白，使核心交互内容（社交链接）位置上移。
- **品牌一致性**: 所有的页面元素（个人信息、社交账号、友情链接）都统一在“卡片”这一物理隐喻下。
